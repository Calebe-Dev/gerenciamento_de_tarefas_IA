Visão geral

Dois usos, um repositório

Esse projeto serve tanto para aprender o fluxo quanto para aplicá-lo direto no seu código.

Base de conhecimento

Guias passo a passo de como instalar, autenticar (sem API key) e usar cada agente no dia a dia — Claude Code, Codex e Antigravity.

Template reutilizável

Issue templates, CLAUDE.md, mapa de arquitetura e memória persistente entre sessões (PROGRESS.md) — copie para qualquer projeto com um único prompt.

O fluxo, resumido

  • Backlog organizado num GitHub Project (board)
  • Issues padronizadas: Epic, Refactor, Bug, Test, Feature
  • Você abre, localmente, o agente mais adequado para a task
  • O agente lê CLAUDE.md + docs/AGENTS.md + docs/PROGRESS.md, implementa, testa e abre PR
  • Você revisa e dá merge — nunca o agente

A personalidade do projeto

Conheça o Woz

Todo projeto precisa de alguém cuidando do backlog — decidindo o que entra, quebrando trabalho grande em pedaços do tamanho de uma tarde, e mantendo as issues claras o suficiente para qualquer agente pegar e rodar. Aqui esse papel tem nome: Woz, em homenagem a Steve Wozniak — o engenheiro que prefere abrir o cofre e entender a máquina a escrever um relatório de 40 páginas sobre ela.

"Faça simples. Se não cabe numa tarde, quebre até caber." — como o Woz pensa

Woz não é uma quarta ferramenta. É uma persona de Product Owner que qualquer um dos três agentes veste quando o trabalho é organizar o backlog em vez de implementar:

prompt do Woz 
Aja como o Woz (docs/WOZ.md) e [tarefa de backlog]

Na prática, ele quebra Epics (type:epic) em sub-issues usando o suporte nativo de sub-issues do GitHub (gh issue create --parent).

O que o Woz faz

  • Triagem — labels (type:*, scope:*, agent:*) e move issues para "Ready"
  • Refinamento — preenche lacunas em issues vagas usando os templates como checklist
  • Quebra de Epics — transforma type:epic em sub-issues menores
  • Anti-bloat — sinaliza scope:L e propõe quebrar em S/M
  • Higiene do board — fecha duplicatas e linka dependências

Instale em qualquer projeto

Copie o prompt abaixo e cole no Claude Code, Codex ou Antigravity, na raiz do projeto que você quer organizar. O agente clona este repositório, copia os arquivos certos e preenche tudo sozinho.

   revisão manual.

No dia a dia

Fluxo do dia a dia

Do backlog ao merge — tudo via gh CLI, sem bots de CI e sem API key.

0. Woz faz grooming (periódico)

Refina issues vagas, quebra Epics em sub-issues e garante que tudo tem type:* e scope:*.

1

Criar issue

gh issue create --template feature.yml (ou pela UI) → cai em "Backlog"

2

Triage

Você adiciona labels de type/scope/agent → item vai para "Ready"

3

Escolher agente

Abra, na sua máquina, o agente indicado pela label agent:* (Claude Code, Codex ou Antigravity)

4

Agente executa

Lê CLAUDE.md + docs/AGENTS.md + docs/PROGRESS.md, implementa, testa e abre PR

5

Review

Você (ou outro agente) roda gh pr diff e revisa — item move para "Review"

6

Merge

gh pr merge --squash --delete-branch → issue fecha, item vai para "Done"

Antes de começar

Boas práticas

Faça

  • Mantenha o CLAUDE.md curto e atualizado
  • Atualize o docs/PROGRESS.md ao final de cada task, com o agente que executou
  • Use labels scope:S/M/L para quebrar tasks grandes
  • Rode uma rodada do Woz periodicamente para manter o backlog organizado
  • Revise sempre os PRs gerados — você é o árbitro final, nunca o agente

Evite

  • Issues vagas tipo "melhorar performance" — sempre especifique escopo
  • Misturar múltiplas mudanças numa issue só
  • Dar permissão de merge direto pro agente
  • Esquecer de atualizar o docs/PROGRESS.md
